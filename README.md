# Prompt Library — Trading & AI Automation

Библиотека промптов для трейдинга и работы с Claude.

**Автор:** Vasilis Golda
**Создана:** Май 2026
**Принцип:** каждый промпт улучшается из практики, не из теории

---

## Структура

```
prompt-library/
├── universal/
│   ├── anti_sycophancy.md      ← защита от sycophancy, стресс-тест (любой проект)
│   └── structured_approach.md  ← анализ перед любым действием (любой проект)
├── trading/
│   ├── signal_request.md       ← запрос торгового сигнала
│   ├── morning_analysis.md     ← утренний анализ рынка
│   ├── position_review.md      ← разбор открытой позиции
│   ├── trade_debrief.md        ← разбор закрытой сделки
│   └── weekly_review.md        ← недельный обзор результатов
├── research/
│   ├── pattern_discovery.md    ← поиск новых паттернов
│   ├── strategy_stress_test.md ← стресс-тест стратегии
│   └── fundamental_context.md  ← анализ новостного контекста
├── agents/                     ← системные промпты агентов (заполняется позже)
└── meta/
    ├── prompt_template.md      ← шаблон для нового промпта
    ├── prompt_review.md        ← оценка и улучшение промпта
    └── working_with_claude.md  ← личные принципы работы с Claude
```

**Принцип разделения:**
- `universal/` — работает в любом проекте и домене
- `trading/` — специфика золота, MGC, объёмного анализа
- `research/` — исследования и анализ паттернов
- `agents/` — системные промпты для AI агентов
- `meta/` — промпты для работы с самими промптами

## Как пользоваться

Каждый файл содержит: Контекст → Промпт → Пример → Антипаттерны → История версий.

Плейсхолдеры в промптах заменяй реальными данными перед отправкой.

---

## 5 принципов эффективных промптов

Выработаны в процессе практики:

1. **Конкретность beats абстрактность** — числа, уровни, примеры дают лучший результат чем общие описания
2. **Few-shot примеры** — реальные примеры из практики передают неформализованное знание которое сложно объяснить словами
3. **Независимость биаса** — явно указывать агенту формировать мнение независимо от твоего описания
4. **Условия отмены** — всегда включать когда сценарий становится недействительным
5. **Стресс-тест** — просить агента найти аргументы против своей же рекомендации

## Принципы обновления

- Если промпт дал отличный результат — зафиксируй **что именно** сработало
- Если результат слабый — запиши **что изменил и почему**
- Каждая версия промпта лучше предыдущей на основе реального использования
- Статусы: `Draft` → `Active` → `Deprecated`
